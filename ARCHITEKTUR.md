# Tabellenarten

## Stammdaten

Eigenständige Objekte.

Beispiele:

- Mitglieder
- Mannschaften
- Plätze
- Bereiche
- Sportanlagen

## Beziehungstabellen

Verbinden zwei Stammdaten.

Beispiele:

- TEAM_MEMBERS
- MEMBER_ROLES

Regel:

Keine Fremdschlüssel in Stammdatentabellen,
wenn eine Beziehung mehrfach vorkommen kann.

Beziehungen werden immer über eigene Tabellen gespeichert.

Regel 8

Fachliche Warnungen sind besser als technische Verbote.

Beispiele:

⚠ Spieler bereits in 3 Mannschaften

⚠ Trainer betreut bereits 4 Teams

⚠ Schiedsrichterkosten fehlen

⚠ Mannschaft hat keinen Trainer

Die Software weist darauf hin, aber der Benutzer entscheidet.
